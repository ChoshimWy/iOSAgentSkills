# 任务清单 / PR Checklist HTML 模板

## 默认风格
- Notion 风格优先：浅色、克制、目录清晰、 checklist 易扫读。

## 顶部规范
- 标题下方的生成日期、来源、作者、仓库等信息必须拆成独立行。
- 如果来源过长，可单独换行或用代码样式展示，但不要和日期 / 作者拼接在同一行。

## 结构
- Hero：标题、适用范围、排期口径
- 全局前置检查
- 阶段总览
- 分阶段任务清单
- 依赖与阻塞条件
- 验收标准
- PR 拆分建议
- 最终门禁

## 每个阶段必须写清
- 目标
- 阻塞依赖
- 主要文件 / 模块
- 开发任务
- 完成判定
- 测试 / 验证要求

## 推荐展示方式
- 阶段任务用 checklist
- 依赖用 callout
- 里程碑 / PR 划分用 table
- 门禁与回归用独立 section

## Checklist 状态规范
- 任务清单中的每一项必须带状态标识。
- 已完成：使用 `√`，建议通过 `.done` 或等价状态类呈现。
- 未完成 / 待办：使用 `□`，作为默认状态。
- 若需要额外表达进行中，可在状态文字里补充，但不要移除 `√ / □` 基础标识。
