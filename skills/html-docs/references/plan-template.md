# 方案 / 提案 HTML 模板

## 默认风格
- Notion 风格优先：浅色优先、轻边框、正文优先、目录清晰；最终 HTML 必须按 `dark-mode-style.md` 支持系统暗黑模式。

## 结构
- Hero：标题、创建日期、更新日期、来源、作者、适用范围、状态（元信息逐行展示）
- 目录：自动锚点跳转
- 背景与问题
- 目标与原则
- 方案总览
- 详细设计
- 接口 / 数据模型 / 时序
- 风险与回退
- 工时 / 里程碑（如适用）
- 总结

## 必须包含
- 结论先行
- 问题与目标明确区分
- 架构 / 数据 / 同步 / 播放 / UI 边界说明
- 风险与回退路径
- 需要时给出接口请求 / 返回示例

## 示例区块建议
- 用 callout 表示“核心结论”
- 用 table 表示“模块拆分 / 对比 / 风险矩阵”
- 用 pre/code 表示“JSON / 命令 / 配置”

## 顶部规范
- 标题下方先给一行主说明，再分行展示创建日期、更新日期、来源、作者、仓库或版本信息。
- 不要把多个元信息写进同一段落，用 `｜` 仅在极简展示场景下临时使用。

## 文件名规范
- 方案 HTML 文件名默认不带日期。
- 若需要版本区分，优先通过顶部“创建日期 / 更新日期”表达，而不是文件名追加日期。
